# 支持用名称自动推断模型提供商

![[LangChain自动推断模型的提供商不需要手动导入.png|860]]

# init_chat_model

**把「模型供应商」当成参数传入**，而不是让你去导入具体的类。

```python
from langchain.chat_models import init_chat_model

# 1. 初始化 OpenAI 模型
gpt_model = init_chat_model("gpt-4o", model_provider="openai", temperature=0)

# 2. 切换到 Ollama 本地模型（代码结构完全一致）
local_model = init_chat_model("qwen2.5-7b", model_provider="ollama", temperature=0)
```

- **极易解耦与配置化**：你只需要修改 `"model_provider"` 和 `"model"` 这两个字符串参数，就能在 OpenAI、Anthropic、Ollama 之间无缝切换。非常适合把模型参数写进 `.env` 或 `config.yaml` 配置文件中。

- **自动导包**：你不需要在代码顶部写一堆 `from langchain_openai import ChatOpenAI` 这样的供应商特定导包语句，LangChain 会在底层动态加载对应的依赖库（前提是你已经 `pip install` 了对应的集成包）。

- **推断能力**：如果你省略 `model_provider`，它甚至能根据常见的模型名称（如 `gpt-4` 或 `claude-3`）自动推断出供应商。
	- ![[基于模型名称推断模型提供商.png|782]]
	- 不足：支持的模型有限，官网：https://docs.langchain.com/oss/python/integrations/providers/overview

# Model 类

这是最经典的初始化方式。每个模型供应商都有一个专门的 Python 类，直接对应其底层的 API 适配器。

```python
# 需要导入特定供应商的专属包
from langchain_openai import ChatOpenAI
from langchain_community.chat_models import ChatOllama

# 1. 初始化 OpenAI 类
gpt_model = ChatOpenAI(model="gpt-4o", temperature=0)

# 2. 初始化 Ollama 类
local_model = ChatOllama(model="qwen2.5-7b", temperature=0)
```

- **原生参数支持与代码提示**：由于是专门的类，IDE（如 PyCharm/VS Code）可以完美提供该供应商专属参数的代码补全（例如某些特定云厂商独有的 `api_base`、`region` 等参数）。
- **直观显式**：代码肉眼可见地明确了当前使用的是哪家服务，对于不需要频繁更换底层模型的小型专用项目，这种写法更直接。
