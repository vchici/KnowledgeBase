# 请求体参数，很多，具体可查文档

【model模型】名称切换：思考/非思考模式

【message消息】role与content组成的对象数组
- 【role为system时】content的作用：身份切换，业务说明，对话背景
- 【role为user时】content的作用：用户的提问

【stream调用方式】
- bool值，表示模型是流式/阻塞调用

【temperature】
- 控制模型生成多样性，值更大，随机性更强

【tools】
- 包含一个或多个工具对象的数组

# 大模型记忆功能

模型API是无状态的API：服务端不记录用户上下文，需要每次请求中显式传入历史对话消息

实现多轮对话的核心是维护一个message数组，将用户提问和模型的回答追加到数组中，作为下一次请求的输入

# LangChain

所有都是围绕Agent提供的便捷开发能力，作用是
1. 通过接口组装请求体参数，比如将用户输入的message和定义的工具信息组装起来，发送给大模型
2. 分析模型返回的结果，帮助调用工具

## Agent执行流程

用户提问 ---> 模型分析(Reasoning) ---> 调用工具 ---> 分析工具执行结果(Observation) ---> 输出

Reasoning（分析需求）:
1. 是否需要调用工具
2. 调用什么工具

调用工具:
模型本身没有调用工具的能力，但是返回的信息可以包含想要调用工具的信息，通过分析这个返回的结果，帮助模型调用工具（LangChain做的）

Observation（感知分析）:
1. 工具执行结果是否满足回答用户问题
