# git log

**缺点：** 信息量大，默认会用一整页显示提交哈希（SHA-1）、作者、日期和完整的提交说明，连续按回车或空格翻页，按 `q` 退出。

# git log -n

是用来**限制输出的提交（Commit）数量**的参数。其中的 `n` 代表一个具体的**正整数**。

简单来说，就是：“**请帮我列出最近的 n 条提交记录，再久远的就不用看了。**”

你既可以写连字符 `-n`，也可以直接省略掉字母 `n`，直接在连字符后面写数字（后者在日常开发中更常用）。

`git log -n 3` 完全等价 `git log -3`，但是工业界编写脚本时都会严格使用 `-n 数量` 这种形参和实参分离的写法。

# git log --oneline

**效果：** 将每个提交压缩到一行，只显示 **前7位哈希值** 和 **提交信息的第一行**。非常适合快速浏览历史。

## git log --oneline --graph main dev

查看 main 分支和 dev 分支的所有日志，当分支记录很多时，屏幕会进入阅读模式：
- 按 **`J` / `K`** 键上下逐行滚动。
- 按 **`Q`** 键（Quit）随时退出查看，返回正常的终端输入状态。

# 查看某个特定开发者提交的代码：

`git log --author="张三"`

# 查看最近两周的提交：

`git log --since="2 weeks ago"`

# 查看特定时间段（某天到某天）：

`git log --after="2026-05-01" --before="2026-05-15"`

`git log --after="2026-05-01"`

# 查找提交信息里包含 "fix" 或 "bug" 的记录：

`git log --grep="fix"`

# 只想看某个特定文件（或模块目录）的修改历史：

`git log -- src/main.js`
