# 问题

海量请求同时发现缓存失效，涌入后端数据库加载数据并回设缓存。

# 解决思路

要么不让热点Key过期，要么不让海量请求同时打入数据库。

## 不让海量请求打入数据库

### 同步互斥锁

如果缓存中已有数据，则返回，若无，线程抢夺同一把锁；抢到后双检，若缓存已被回填则返回缓存值，若未，则读数据库然后回填缓存；没有抢到的，休眠一段时间重试。

优点：一致性（返回的数据永远是最新的数据）
确定：并发吞吐量低，线程会等待，有死锁风险，需要设置锁过期时间，避免过长时间持有锁

```java
public String getData(String key) {
    // 1. 从Redis获取数据
    String value = redis.get(key);
    if (value != null) {
        return value;
    }
    
    // 2. 缓存失效，尝试获取分布式锁
    String lockKey = "lock:" + key;
    if (redis.setnx(lockKey, "1", 30L)) { // 锁设置30秒过期，防止死锁
        try {
            // 双重检查(Double Check)，防止在获取锁的过程中别人已经生成了缓存
            value = redis.get(key);
            if (value != null) return value;
            
            // 3. 查询数据库
            value = db.query(key);
            // 4. 回写缓存
            redis.set(key, value, 3600L); 
        } finally {
            // 5. 释放锁
            redis.del(lockKey);
        }
    } else {
        // 6. 没拿到锁的线程，休眠一段时间后重试
        Thread.sleep(50);
        return getData(key); // 递归重试
    }
    return value;
}
```
### 异步互斥锁

没有物理过期时间，逻辑过期时间写在数据中，若拿到的数据发现是过期的，先返回，然后获取互斥锁，获取后开启异步线程读数据库更新缓存，没拿到锁的直接返回旧数据

优点：高响应（拿到数据无需等待）
缺点：弱一致性，异步更新完成前，拿到的都是旧数据

```java
public RedisData getData(String key) {
    // 1. 从Redis获取数据
    String json = redis.get(key);
    RedisData redisData = JSON.parse(json);
    
    // 2. 判断是否逻辑过期
    if (redisData.getExpireTime().isAfter(LocalDateTime.now())) {
        // 未过期，直接返回
        return redisData;
    }
    
    // 3. 已过期，尝试获取互斥锁
    String lockKey = "lock:" + key;
    if (redis.setnx(lockKey, "1", 10L)) {
        // 拿到锁，开启独立线程去异步更新
        executorService.submit(() -> {
            try {
                // 查询DB并写入Redis（包含新的逻辑过期时间）
                rebuildCache(key);
            } finally {
                redis.del(lockKey);
            }
        });
    }
    // 4. 没拿到锁或者拿到锁的线程，都直接返回历史旧数据
    return redisData;
}
```

### Sentinel / 线程池限流

针对访问该热点Key的接口设置**本地线程池大小**或**QPS阈值**

放行少量线程查DB

## 不让热点Key过期

### 不设置过期时间

### 监控，续期

热点Key即将到期前，提前从数据库刷一次数据回缓存。

# 应用场景

金融、库存、抢购： _一致性要求高_

资讯、视频播放量：_允许微小延迟_

已知爆款、常驻热点：_手动预热 + 永不过期_
