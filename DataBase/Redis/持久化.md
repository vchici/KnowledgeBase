# RDB内存快照

RDB（Redis Database）被指定时间间隔内，写入磁盘，过程是 bgsave，异步后台执行。

bgsave 是通过 fork() 子进程异步创建 RDB 文件的，Linux 的 fork() 采用了 COW（copy on write）

## copy on write

在 Linux 中，一个内存页默认是4KB

Redis 进行 bgsave 创建子进程时，子进程拷贝的是主进程的“页表”（内存地址引用目录）

当操作系统发现，主线程要修改 物理内存中 第100号 数据页，子线程要读取 第100、101号 数据页，因为Redis内存中，有一只大Hash键值对占用了这两个页，此时触发COW机制

把 100号 数据页（4KB）完整拷贝到一份新的物理内存 第999号 页，然后让主进程改 999号 页，子线程继续读取没变的 100号 页，来写RDB文件

即使 100号页 只有1字节的改动，但是造成了 4KB 的内存拷贝

### 预防内存超限

#### 配置 Linux 的 `vm.overcommit_memory = 1`（核心）

内核参数设为 `1`，意思是允许乐观分配内存，RDB期间不可能所有的key都被修改（10%～20%频繁变动的热点数据），允许透支分配，防止fork失败导致Redis阻塞

#### 控制Redis单实例大小

单台Redis内存控制8～10GB

数据量大用Redis集群

#### 主从分离

[主从](主从、哨兵.md)

主节点不做RDB，全部性能留给高并发读写

选一个从节点专门用来做RDB备份，即使因为COW导致内存翻倍，不影响线上核心业务
