# 问题

1. 海量请求同时发现大量Key缓存失效，涌入后端数据库加载数据并回设缓存。
2. Redis发生宕机

# 解决思路

## 海量Key缓存失效

避免大量Key拥有完全相同的 `TTL` 

对于极度热点的数据，设置缓存不过期（不设置`TTL`）

互斥锁，如果缓存失效，通过分布式锁或本地锁，只允许一个线程读数据库并写回缓存

## Redis宕机

### 哨兵模式

实现主从切换，保证主节点宕机后有新主节点顶上

### 集群模式

数据分片存储，

### 多级缓存架构

在应用服务器本地再增加一级缓存

请求先读本地缓存 -> 没命中读Redis缓存 -> 没命中读数据库

如果Redis集群整体宕机，本地缓存抗住大部分流量

### 熔断与降级

大量流量逼近数据库时，熔断对数据库的访问

应用执行降级逻辑，返回默认值、友好的提示错误页
