存储检索高维向量的数据库

常用产品：
- Pinecone、Milvus、Weaviate
- Chroma、Qdrant、Faiss
- pgvector（PostgreSQL扩展）

传统数据库
查询类型	精确匹配、范围查询
索引结构	B+树、哈希
查询方式	WHERE id = 1
返回结果	完全匹配的记录
存储对象	结构化数据

向量数据库
查询类型	相似度搜索（ANN）
索引结构	HNSW、IVF、PQ
查询方式	query(vector, top_k=10)
返回结果	最相似的k个向量
存储对象	Embedding向量

# ANN算法
 
Approximate Nearest Neighbor

近似最近邻搜索，在牺牲少量精度的情况下大幅提升查询速度

不遍历所有向量，而是通过索引结构快速定位可能相似的候选集

典型算法：HNSW（层次化可导航小世界图）、IVF（倒排索引）、PQ（乘积量化）

# Embedding

将文本/图像转换为固定长度的数值向量

向量在高维空间中表达语义，相似内容距离近

什么时候需要
语义搜索（用户输入找相似文档）
RAG（大模型检索知识库）
推荐系统（找相似内容）
聚类分析、去重
