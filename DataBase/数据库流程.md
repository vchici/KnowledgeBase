$$
\text{代码业务请求} \xrightarrow{\text{连接池}} \text{SQL 预编译} \xrightarrow{\text{网络传输}} \text{数据库引擎执行 (事务/索引)} \xrightarrow{\text{结果集}} \text{ORM 映射为对象}
$$

# 连接池复用长连接

应用与数据库之间通过 TCP 协议建立长连接，用连接池复用

性能好，后续SQL传输不需要重新TCP握手

事务支持，分布式事务、锁机制，依赖底层TCP驱动协议（如MySQL协议）

# HTTP连接池连接Serverless数据库

应用向数据库发送一个标准的标准 HTTPS 请求，Body 里面带上 SQL 或查询条件。

数据库网关（Gateway）在云端自己维护连接池，前端的 HTTP 请求随落随走（无状态），完美解决了 Serverless 频繁启停的问题。
