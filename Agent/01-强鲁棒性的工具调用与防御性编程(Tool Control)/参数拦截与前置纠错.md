**参数拦截与前置纠错**：

- **学习重点**：编写专门的“参数修复 Prompt”或在 Python 后端建立静态硬编码映射。
    
- **核心痛点**：大模型调用工具时极易发生“幻觉”（比如编造一个不存在的参数名）。在工具真正执行前，必须有一层代码对其进行严格的语法与语义检查。

# field_validator model_validator

| **模式**              | **field_validator 接收到的 v** | **model_validator 接收到的 v**     |
| ------------------- | -------------------------- | ------------------------------ |
| **`mode='before'`** | 该字段的原始输入值                  | 整个模型的**原始输入字典（`dict`）**        |
| **`mode='after'`**  | 该字段解析后的安全值                 | 已经实例化好的 **Model 对象本身（`self`）** |
| **`mode='wrap'`**   | 原始输入值 + `handler`          | **原始输入字典（`dict`）** + `handler` |

如果你的校验逻辑**同时牵扯到多个字段**（也就是字段之间有联动、联合制约关系），`field_validator` 就很难搞定了，这时候必须请 `model_validator` 出山。
