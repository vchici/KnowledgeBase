我们可以用一个“去办事大厅办事”的例子，彻底理清 **Tomcat、Servlet、JSP / JSON** 三者的物理关系：

- **Tomcat 是“办事大厅的物理建筑”：** 它负责开门、装空调、拉网线（监听端口、维护 TCP 连接）。但是它自己不会办业务。

- **Servlet 是“柜台里的业务员”：** 当一个市民（浏览器请求）冲进大厅，Tomcat 会根据你办的业务（URL），把你领到对应的柜台。**那个坐在柜台后面、真正听你诉求（解析参数）、帮你翻档案（查数据库）、给你办业务的 Java 类，就是 Servlet。**

- **JSP / JSON 是“办完业务给你的回执”：** 业务员把业务办好了，打印出一张表格（JSP 渲染成 HTML）或者给你一串代码（JSON 报文），让你带回家。

Servlet（Server Applet）就是一段运行在服务器（Tomcat）上的、专门用来处理网络请求的 Java 程序。
