> 定义一个统一的算法接口，把每种不同的业务方案封装成独立的类，运行时自由替换。
# 定义策略接口

```java
// 1. 定义策略接口
interface PaymentStrategy {
    double calculate(double amount);
}
```

# 具体策略类

```java
// 2. 具体的策略类 A：满减
class CashbackStrategy implements PaymentStrategy {
    public double calculate(double amount) { return amount - 20; }
}

// 3. 具体的策略类 B：打折
class DiscountStrategy implements PaymentStrategy {
    public double calculate(double amount) { return amount * 0.8; }
}
```

# 环境类

```java
// 4. 环境类（中间层）：用来接收和执行策略
class PriceCalculator {
    private PaymentStrategy strategy;

    public void setStrategy(PaymentStrategy strategy) { this.strategy = strategy; }

    public double animateCalculate(double amount) {
        // 多态：具体执行哪个算法，看运行时传入的是谁
        return strategy.calculate(amount);
    }
}
```
