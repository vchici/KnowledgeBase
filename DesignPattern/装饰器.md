> 一个实现了接口的类，内部又包含了这个接口的对象，并以此来增强该对象的功能。
# 被装饰类接口 Coffee

`interface Coffee {}`

`class Espresso implements Coffee` Espresso是Coffee的纯实现类

# (抽象)装饰器基类 CondimentDecorator 实现 Coffee

表示 被装饰之后 的类 还是被装饰类的一种，通过**构造方法传递要被装饰的对象**。

```java
abstract class CondimentDecorator implements Coffee {
	// 可以是被装饰过的Coffe或者就是Coffe
	protected Coffee decoratedCoffee;
	public CondimentDecorator(Coffee coffee) { 
		this.decoratedCoffee = coffee; 
	}
}
```

这里是否需要抽象取决于加料方式，如果只有一种，可以不抽象，用普通实现类就行。
# 具体装饰器 Milk 继承 CondimentDecorator

```java
class Milk extends CondimentDecorator {
	public Milk(Coffee coffee) {
		super(coffee);
	}
}
class Mocha extends CondimentDecorator {
    public Mocha(Coffee coffee) {
        super(coffee);
    }
}
```

# 客户端调用

```java
class Main {
	public static void main(String[] args) {
		Coffee myCoffee = new Espresso(); // Coffee的实现类Espresso
		myCoffee = new Milk(myCoffee); // 加奶
		myCoffee = new Mocha(myCoffee); // 加摩卡
	}
}
```
