> 提供一个接口，用于创建**相关或依赖对象的家族**，而不需要明确指定具体类。
# 产品家族

多个产品之间有配套关系，比如 Windows 风格的按钮和文本框，Mac 风格的按钮和文本框：

```java
// 产品A：按钮
interface Button {
    void click();
}
class WindowsButton implements Button {
    public void click() { System.out.println("Windows按钮点击"); }
}
class MacButton implements Button {
    public void click() { System.out.println("Mac按钮点击"); }
}

// 产品B：文本框
interface TextField {
    void input();
}
class WindowsTextField implements TextField {
    public void input() { System.out.println("Windows文本框输入"); }
}
class MacTextField implements TextField {
    public void input() { System.out.println("Mac文本框输入"); }
}
```

# 抽象工厂接口

```java
// 抽象工厂：定义创建一族产品的方法
interface GUIFactory {
    Button createButton();
    TextField createTextField();
}
```

# 具体工厂

```java
// Windows工厂：只生产Windows家族的产品
class WindowsFactory implements GUIFactory {
    public Button createButton() {
        return new WindowsButton();
    }
    public TextField createTextField() {
        return new WindowsTextField();
    }
}

// Mac工厂：只生产Mac家族的产品
class MacFactory implements GUIFactory {
    public Button createButton() {
        return new MacButton();
    }
    public TextField createTextField() {
        return new MacTextField();
    }
}
```

# 客户端调用

```java
class Application {
    private Button button;
    private TextField textField;
    
    // 注入工厂，客户端不知道具体是哪个家族
    public Application(GUIFactory factory) {
        this.button = factory.createButton();
        this.textField = factory.createTextField();
    }
    
    public void run() {
        button.click();
        textField.input();
    }
}

class Main {
    public static void main(String[] args) {
        GUIFactory factory = getFactoryFromConfig();  // 外部决定用哪个家族
        Application app = new Application(factory);
        app.run();  // 输出对应家族的产品行为
    }
    
    static GUIFactory getFactoryFromConfig() {
        String os = System.getProperty("os.name");
        return os.contains("Windows") ? new WindowsFactory() : new MacFactory();
    }
}
```

# 与工厂方法的区别

| 工厂方法 | 抽象工厂 |
|---------|---------|
| 创建**一种**产品 | 创建**一族**产品 |
| 一个工厂只一个create方法 | 一个工厂多个create方法 |
| 子类决定实例化哪个类 | 工厂保证产品家族一致性 |

抽象工厂保证：**同一个工厂创建的产品是配套的，不会出现 Windows 按钮配 Mac 文本框的情况**。
