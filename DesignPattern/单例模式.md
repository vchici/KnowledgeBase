> 一个类在整个程序运行期间只能有一个实例，并提供一个全局访问点。
# 私有构造方法

```java
class Singleton {
    // 1. 私有构造方法，防止外部new对象
    private Singleton() {}
    
    // 2. 静态变量保存唯一实例
    private static Singleton instance;
    
    // 3. 公开的静态方法获取实例
    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
```

# 线程安全的单例（双重检查锁）

```java
class Singleton {
    private static volatile Singleton instance;
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {                    // 第一次检查
            synchronized (Singleton.class) {
                if (instance == null) {            // 第二次检查
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

# 双重检查锁的关键点

**第一次检查**：避免不必要的锁竞争。如果实例已存在，直接返回，不用进同步块，提升性能。

**synchronized**：保证只有一个线程能进入创建实例的代码块，防止多线程同时创建多个实例。

**第二次检查**：防止并发问题。线程A和线程B同时通过了第一次检查，线程A先获得锁并创建了实例，释放锁后线程B进入同步块，如果没有第二次检查，B会再次创建实例。

**volatile**：防止指令重排序。`new Singleton()` 不是原子操作，分三步：
1. 分配内存空间
2. 初始化对象
3. 将引用指向分配的内存

JVM可能重排序为1→3→2，导致其他线程在第一次检查时看到instance不为null（已指向内存），但对象还没初始化（步骤2未完成），就会返回一个未初始化的对象。volatile保证1→2→3的顺序执行。

# 客户端调用

```java
class Main {
    public static void main(String[] args) {
        Singleton s1 = Singleton.getInstance();
        Singleton s2 = Singleton.getInstance();
        // s1 == s2 返回 true，始终是同一个对象
    }
}
```
