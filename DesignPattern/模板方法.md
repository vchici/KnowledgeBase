> 父类写死流程的骨架，把个别多变的步骤留空，让子类去填空。
# 抽象类模板 CookTemplate

`abstract class CookTemplate`

# final 修饰模板方法 不允许子类重写

```java
public final void cookProcess() {
	pourOil();
	heatOil();
	pourIngredient(); // 填空题 1
	addCondiments();   // 填空题 2
	fryAndOut();
}
// （固定行为，子类直接继承即可）
private void pourOil() {}
private void heatOil() {}
private void fryAndOut() {}
// （变化点：留给子类去实现）
protected abstract void pourIngredient();
protected abstract void addCondiments();
```

# 子类实现 CookTemplate

```java
// 具体实现类 A：炒西红柿鸡蛋
class CookTomatoEgg extends CookTemplate {
    @Override
    protected void pourIngredient() {
        System.out.println("3. 倒入切好的西红柿和打散的鸡蛋");
    }

    @Override
    protected void addCondiments() {
        System.out.println("4. 加入两勺盐和一勺糖");
    }
}
// 具体实现类 B：炒手撕包菜
class CookCabbage extends CookTemplate {
    @Override
    protected void pourIngredient() {
        System.out.println("3. 倒入洗净撕碎的包菜叶，加点干辣椒");
    }

    @Override
    protected void addCondiments() {
        System.out.println("4. 沿着锅边淋入两勺生抽、一勺香醋");
    }
}
```
