> 定义一个创建对象的接口，让子类决定实例化哪个类，将对象的创建延迟到子类中。
# 产品接口

```java
// 1. 产品接口
interface Product {
    void use();
}
```

# 具体产品

```java
// 2. 具体产品 A
class ProductA implements Product {
    public void use() {
        System.out.println("使用产品A");
    }
}

// 3. 具体产品 B
class ProductB implements Product {
    public void use() {
        System.out.println("使用产品B");
    }
}
```

# 工厂接口与具体工厂

```java
// 4. 工厂接口
interface Factory {
    Product create();
}

// 5. 具体工厂 A：只负责创建 A
class FactoryA implements Factory {
    public Product create() {
        return new ProductA();
    }
}

// 6. 具体工厂 B：只负责创建 B
class FactoryB implements Factory {
    public Product create() {
        return new ProductB();
    }
}
```

# 客户端调用

```java
// 客户端只依赖接口，不依赖具体实现
class Client {
    private Factory factory;
    
    // 通过构造方法注入工厂（具体是A还是B，由外部决定）
    public Client(Factory factory) {
        this.factory = factory;
    }
    
    public void doSomething() {
        Product product = factory.create();  // 客户端不知道创建的是谁
        product.use();
    }
}

class Main {
    public static void main(String[] args) {
        // 由外部配置决定用哪个工厂
        Factory factory = getFactoryFromConfig();  
        Client client = new Client(factory);
        client.doSomething();  // 客户端只管用，不知道具体产品
    }
    
    static Factory getFactoryFromConfig() {
        String type = System.getProperty("product.type");
        return "A".equals(type) ? new FactoryA() : new FactoryB();
    }
}
```

关键点：**客户端只接收 Factory 接口，具体用哪个工厂由外部配置/注入决定**。新增 ProductC 时，只需新增工厂类，客户端代码无需修改。

# 简单工厂（静态工厂）

如果产品种类少且不常变动，可以用简单工厂：

```java
class SimpleFactory {
    public static Product create(String type) {
        if ("A".equals(type)) {
            return new ProductA();
        } else if ("B".equals(type)) {
            return new ProductB();
        }
        return null;
    }
}
```

缺点：新增产品需要修改工厂代码，违反开闭原则。
