> 新接口的实现类中，包含旧类的对象，并在实现新方法时去调用旧方法。
# OldLogger NewLogger

```java
// 1. 新系统定义的标准接口（目标接口）
interface NewLogger {
    void printLog(String message);
}

// 2. 老项目的旧类（被适配者：Adaptee）
// 它有我们想要的实际功能，但是接口（方法名）对不上
class OldLogger {
    public void logSpecificMessage(String msg) {
        System.out.println("[旧日志组件核心逻辑] 正在记录: " + msg);
    }
}
```

# LogAdapter

```java
// 3. 适配器（Adapter）：两头牵线搭桥
class LogAdapter implements NewLogger {
    
    // 内部持有一个旧类的引用
    private OldLogger oldLogger;

    public LogAdapter(OldLogger oldLogger) {
        this.oldLogger = oldLogger;
    }

    @Override
    public void printLog(String message) {
        // 新系统以为自己调用的是 printLog()
        // 实际上适配器在内部偷偷调用了旧类的 logSpecificMessage()
        oldLogger.logSpecificMessage(message);
    }
}
```
